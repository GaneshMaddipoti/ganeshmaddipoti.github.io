Docker

Introduction
• Docker is an open platform for developing, shipping and running applications.
• Docker enable you to separate your applications from infrastructure so you can deliver software quickly.
• With Docker, you can manager your infrastructure the same way you manager your applications.
• By using Docker methodologies, we can significantly reduce the delay between development and running in production.

Docker Platform
• Docker provides the ability to package and run application, in a loosely isolated environment called container.
• Containers are light weight, self-contained, sharable, so scalable, and can run in different host environments.
• Docker provides the tooling and platform to manage the lifecycle of containers.
a. Develop your application and its supporting components using containers.
b. It becomes the unit, distributable, testable and scalable.
c. Once development is ready, deploy it to different environments without any changes.

Use cases
• Fast, consistent delivery of applications.
• Responsive deployment and scalable
• Running more workloads on same h/w

Docker Architecture
• • Docker uses client-server architecture, where docker client talks to docker domain using REST API over UNIX sockets or n/w interface.
• Another docker client is Docker compose, which lets you work with applications consisting of set of containers.

Docker Daemon
○ Docker Daemon (dockerd) listens for Docker API requests and managers Docker objects such as images, containers, n/w and volumes.
○ A Daemon can communicate with other daemons to manage docker services.

Docker Client
○ Docker Client (docker) is the primary way that many docker users interact with Docker.
○ When you run commands like docker run, the client sends these commands to dockerd for execution.
○ The docker command uses the docker API
○ The Docker client can communicate with multiple daemons.

Docker Desktop
○ Docker desktop is easy to install application for MAC, Windows and Linux environments.
○ It enables you to build and share containerize applications and micro services.
○ It includes docker, dockerd, docker compose, docker content trust, Kubernetes, and credential helper.

Docker registries
○ A docker registry stores docker images.
○ Docker is public registry and default when you pull.
○ We can even run our own private registry.

Docker Objects (Images, Containers, Networks, Volumes, Plugins, and other)
Image ○ An image is read only template used to materialize the container.
○ To build an image, we write Dockerfile with simple syntax for defining the steps needed to create an image.
○ Each instruction creates a layer in the image.
○ When you change the Dockerfile and rebuild the image, only those layers which are changed are rebuild.
Container ○ Container is runnable instance of Image.
○ We can create, start, stop, move and delete a container using Docker API or CLI.
○ We can connect a container to more than one networks, attach storage to it, or even create image with current state.

Underlying technology
• Docker is written in Go Programming language.
• Docker uses namespaces to provide isolated workspaces called containers.
